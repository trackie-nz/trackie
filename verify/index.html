Verify Trackie's client code

This page checks that the app code your browser was served is byte-for-byte identical to Trackie's public, open-source build - the exact files produced by the public build pipeline on GitHub, with nothing added or changed.

It runs entirely in your browser and this page is served by GitHub, not by Trackie, so Trackie cannot influence the result. It is a spot check of the one session you capture below.

1. Save your session

Open the app, then open your browser's developer tools (F12) and go to the Network tab. Tick Disable cache, reload the page, click around a little, then right-click any request and choose Save all as HAR.

2. Check it

Loading the published build list…

Drop your .har file here, or click to choose it

What this does and does not prove

It proves: for the session you captured, every piece of code the app served you matches the public build - so no hidden logging or data-stealing code was slipped into your client.

It does not prove: anything about other sessions or the future (re-run it whenever you like); anything about the server's own behaviour; and it does not cover bank-sync data, which by design passes through the server. It applies to end-to-end-encrypted budgets. To trust nobody at all, you can self-host the open-source client yourself.
